Project Overview

This project predicts whether water is safe for drinking using machine learning. It analyzes chemical properties like pH, hardness, and dissolved solids to classify water as potable or non-potable.

Objectives

Analyze water quality parameters
Build ML classification model
Improve prediction accuracy
Improve prediction reliability for real use

Dataset Snapshot

5500 Total Records
2232 Safe (Potable)
3268 Unsafe (Non-potable)

Source used by app: expanded_5500_records.csv. Inputs are 9 water-quality features and output target is Potability (0/1).

Preprocessing & Validation

  • Missing values are handled with median-based filling before graph/model evaluation.
  • IQR clipping is used in analysis views to reduce outlier effect on distributions.
  • Physical range checks run at prediction time (example: pH must be between 0 and 14).
  • Training-range warning is shown when values are outside learned domain.

Model

Random Forest combines multiple decision trees to improve accuracy and reduce overfitting. It works well with complex datasets and provides reliable predictions.

Training & Tuning

  • Compared models: Baseline RF, GridSearchCV RF, and RandomizedSearchCV RF.
  • Best packaged model is loaded from best_water_potability_model.pkl.
  • Conservative safe threshold policy is applied (minimum decision threshold: 44%).
  • Goal: reduce unsafe-to-safe mistakes while keeping practical recall.

Results

83% Accuracy

GridSearchCV improved the performance of the model.

Metrics & Insights

  • Evaluation focus includes Accuracy, Precision, Recall, and error balance.
  • Confusion matrix and model comparison are available in Graphs dashboard.
  • Feature importance and correlation views support explainability.
  • No single feature determines potability; combined signal drives prediction quality.
